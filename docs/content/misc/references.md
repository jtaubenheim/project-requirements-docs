- [Key words for RFCs to indicate requirement levels](https://datatracker.ietf.org/doc/rfc2119/) - Usage of MAY, MUST, SHALL, etc

- ["We do not break userland, period.](https://news.ycombinator.com/item?id=18003084) - Refinement of a Linus Torvalds strong pronouncement that a change to the kernel that breaks a client is a kernel bug. Document references [LKML](https://lkml.org/lkml/2012/12/23/75). Scale can likely adhere to the weaker version of the statement, anything shown to break an API client is a bug.

- [Functional Requirements](https://en.wikipedia.org/wiki/Functional_requirement)
